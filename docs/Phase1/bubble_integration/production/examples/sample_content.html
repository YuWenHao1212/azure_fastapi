🎨 TinyMCE 標記系統 - 示例內容

以下是各種標記類型的示例，可以直接複製到您的 TinyMCE 編輯器中測試。

1. 原有關鍵字 (opt-keyword-existing)

用途：標記履歷中已存在的重要關鍵字

樣式：深藍色背景，白色文字

<span class="opt-keyword-existing">Python</span>

我有 Python 和 JavaScript 開發經驗。

2. 新增關鍵字 (opt-keyword)

用途：標記新添加的關鍵字

樣式：紫色邊框，透明背景

<span class="opt-keyword">React</span>

熟悉 React 和 Vue.js 框架開發。

3. 修改內容 (opt-modified)

用途：標記被修改或優化的內容

樣式：淺黃色背景

<span class="opt-modified">負責系統架構設計和性能優化</span>

在前一份工作中，負責系統架構設計和性能優化，提升了系統處理效率。

4. 新增內容 (opt-new)

用途：標記全新添加的內容區塊

樣式：綠色左邊框

<div class="opt-new">新增的工作經驗或技能描述</div>
雲端架構師 | ABC科技公司 | 2023-2024
負責設計和實施企業級雲端解決方案，包括 AWS、Azure 平台的架構規劃。 成功遷移 15+ 應用程式到雲端，降低營運成本 30%。

5. 佔位符 (opt-placeholder)

用途：可編輯的佔位符，點擊後可以輸入具體數值

樣式：紅色虛線框，斜體

行為：點擊可編輯，自動添加單位

<span class="opt-placeholder">[PERCENTAGE]</span>

提升系統性能 [PERCENTAGE]，管理團隊規模 [TEAM SIZE]。

處理交易金額 [AMOUNT]，服務用戶數量 [USER COUNT]。

6. 改進內容 (opt-improvement)

用途：標記已完成編輯的改進內容

樣式：綠色底線

<span class="opt-improvement">25%</span>

成功提升系統效能 25%，領導 8-10 people 的開發團隊。

7. 完整履歷段落示例

包含多種標記類型的完整示例

<div class="opt-new"> <strong>高級軟體工程師 | XYZ科技 | 2022-2024</strong><br> 負責<span class="opt-modified">大型電商平台的後端開發和架構優化</span>，使用 <span class="opt-keyword-existing">Python</span>、 <span class="opt-keyword-existing">Django</span> 和 <span class="opt-keyword">FastAPI</span> 開發高性能 API。 <br><br> 主要成就：<br> • 優化數據庫查詢，提升系統響應速度 <span class="opt-placeholder">[PERCENTAGE]</span><br> • 領導 <span class="opt-placeholder">[TEAM SIZE]</span> 完成微服務架構重構<br> • 處理日交易量達 <span class="opt-placeholder">[USER COUNT]</span> 筆<br> • 節省服務器成本 <span class="opt-placeholder">[AMOUNT]</span>/年 </div>
高級軟體工程師 | XYZ科技 | 2022-2024
負責大型電商平台的後端開發和架構優化，使用 Python、 Django 和 FastAPI 開發高性能 API。

主要成就：
• 優化數據庫查詢，提升系統響應速度 [PERCENTAGE]
• 領導 [TEAM SIZE] 完成微服務架構重構
• 處理日交易量達 [USER COUNT] 筆
• 節省服務器成本 [AMOUNT]/年

8. Placeholder 類型對應表

以下是系統支援的 placeholder 類型和自動格式化規則：

Placeholder 輸入示例 格式化結果
[PERCENTAGE] 25 25%
[TEAM SIZE] 8 8 people
[AMOUNT] 50000 $50000
[TIME PERIOD] 6 6 months
[USER COUNT] 50000 50.0K

9. 使用說明

  1. 複製 HTML 代碼：點擊各示例的「複製」按鈕
  2. 貼上到編輯器：在 TinyMCE 編輯器中貼上 HTML 代碼
  3. 測試功能：
    • 使用 Toggle 按鈕切換所有標記顯示/隱藏
    • 使用 Checkbox 控制個別標記類型
    • 點擊 placeholder 進行編輯
  4. 確認效果：標記應該有正確的顏色和樣式
